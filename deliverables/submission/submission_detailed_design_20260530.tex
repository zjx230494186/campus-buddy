\hypertarget{ux5f15ux8a00}{%
\subsection{1. 引言}\label{ux5f15ux8a00}}

\hypertarget{ux9879ux76eeux80ccux666f}{%
\subsubsection{1.1 项目背景}\label{ux9879ux76eeux80ccux666f}}

校园搭子平台面向校内学生的结伴需求，覆盖饭搭子、学习搭子、运动搭子、课程组队和创新项目组队等场景。项目目标不是做一个普通聊天工具，而是提供``发布需求、审核、广场筛选、低压力联系、联系方式解锁、评价与信用摘要''的完整闭环，降低陌生同学之间建立合作或同行关系的成本。

\hypertarget{ux5efaux8bbeux76eeux6807}{%
\subsubsection{1.2 建设目标}\label{ux5efaux8bbeux76eeux6807}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  支持学生通过校园邮箱注册登录，并通过身份认证进入核心业务。
\item
  支持已认证学生发布结构化搭子需求，经管理员审核后进入广场。
\item
  支持其他已认证学生浏览广场、查看发布者信用摘要并发起联系。
\item
  支持会话中的低压力沟通、首条消息节制、未读状态、关闭会话和重新发起。
\item
  支持联系方式卡片保存、双方确认后解锁联系方式。
\item
  支持有效会话后的互评与信用摘要展示。
\item
  支持管理员审核身份认证和帖子内容，保障平台基本秩序。
\item
  支持后端服务器、数据库、对象存储、SMTP 邮件发送和 Windows
  桌面端内测包的可运行交付。
\end{enumerate}

\hypertarget{ux9002ux7528ux8303ux56f4}{%
\subsubsection{1.3 适用范围}\label{ux9002ux7528ux8303ux56f4}}

本文描述当前课程提交版已经实现或已经形成稳定接口的系统设计。Qt
桌面端是当前主要客户端；移动端目前具备 API
文档和接入边界，但正式移动客户端不属于当前实现范围。投诉申诉、通知治理、HTTPS
域名、刷新 token 接口等能力属于后续生产化改进。

\hypertarget{ux975eux529fux80fdux6027ux9700ux6c42}{%
\subsubsection{2.3
非功能性需求}\label{ux975eux529fux80fdux6027ux9700ux6c42}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  安全性：核心接口使用 JWT；管理员接口要求 ADMIN
  角色；广场、发帖、联系、评价要求身份认证通过。
\item
  可维护性：后端采用 Spring Boot 分模块组织；数据库结构由 Flyway
  迁移管理；客户端通过 API service 封装后端合同。
\item
  可部署性：后端支持 deploy profile，服务器由 systemd 托管，Nginx
  对外暴露 HTTP API。
\item
  可演示性：Windows 桌面端已打包为内测 zip，后端公网 health 和关键 smoke
  已形成验证记录。
\item
  密钥隔离：数据库密码、JWT secret、OBS AK/SK、SMTP
  授权码只放服务器或项目外私有环境变量，不写入仓库、文档和客户端。
\end{enumerate}

\hypertarget{ux7cfbux7edfux603bux4f53ux67b6ux6784}{%
\subsection{3.
系统总体架构}\label{ux7cfbux7edfux603bux4f53ux67b6ux6784}}

\hypertarget{ux6574ux4f53ux67b6ux6784ux8bf4ux660e}{%
\subsubsection{3.1
整体架构说明}\label{ux6574ux4f53ux67b6ux6784ux8bf4ux660e}}

系统采用客户端到后端 API 的集中式架构。Qt 桌面端和未来移动端只访问 HTTP
API，不直连数据库、不持有 OBS
或数据库密钥。后端负责鉴权、业务规则、数据持久化、文件上传和邮件发送。

\begin{longtable}[]{@{}ll@{}}
\toprule
层次 & 技术与职责 \\
\midrule
\endhead
客户端层 & Qt Widgets 桌面端；未来移动端可复用同一 REST API \\
API 层 & Spring Boot Controller，统一接收 JSON 或 multipart 请求 \\
业务层 & Auth、Post、Contact、Review、Storage 等 Service \\
数据层 & PostgreSQL/H2 配合 JPA Repository；Flyway 管理迁移 \\
文件层 & OBS 对象存储保存认证材料；测试环境可使用内存实现 \\
外部服务 & SMTP 发送校园邮箱验证码；Nginx 提供公网入口 \\
\bottomrule
\end{longtable}

\hypertarget{ux4e3bux6d41ux7a0bux6982ux89c8}{%
\subsubsection{3.3 主流程概览}\label{ux4e3bux6d41ux7a0bux6982ux89c8}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  学生输入校园邮箱并请求验证码。
\item
  学生校验验证码后完成注册和登录，获得 Bearer token。
\item
  学生提交身份认证材料，管理员审核通过后成为 VERIFIED。
\item
  已认证学生创建帖子草稿，补齐字段后提交审核。
\item
  管理员审核通过帖子，帖子进入广场。
\item
  其他已认证学生浏览广场并发起联系。
\item
  双方通过会话沟通，保存联系方式卡片并双向确认解锁。
\item
  双方达到有效会话条件后提交评价，系统更新信用摘要。
\end{enumerate}

接口，因此客户端遇到 401 时应引导重新登录。

消息才可评价；同一关系只评价一次；24 小时内只可修改一次 \\

\subsubsection{5.1
新用户注册、验证码与登录}\label{ux65b0ux7528ux6237ux6ce8ux518cux9a8cux8bc1ux7801ux4e0eux767bux5f55}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  用户输入 \texttt{bjtu.edu.cn} 校园邮箱。
\item
  客户端调用发送验证码接口，按钮进入倒计时状态。
\item
  用户输入验证码，客户端调用校验接口并获得 verificationTicket。
\item

\item
  用户调用登录接口获得 accessToken。
\item
  客户端根据返回的 authenticationStatus 判断是否进入身份认证页。
\end{enumerate}

\hypertarget{ux8eabux4efdux8ba4ux8bc1ux5ba1ux6838}{%
\subsubsection{5.2
身份认证审核}\label{ux8eabux4efdux8ba4ux8bc1ux5ba1ux6838}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  学生上传认证材料，获得 attachmentId。
\item
  学生填写真实姓名、学号、学院、专业、年级并提交认证。
\item
  认证状态变为 PENDING\_REVIEW。
\item
  管理员在审核页查看队列和材料摘要。
\item
  管理员通过后，用户 authenticationStatus 变为 VERIFIED。
\item
  管理员驳回后，用户可查看原因并重新提交。
\end{enumerate}

\hypertarget{ux53d1ux5e16ux5ba1ux6838ux548cux5e7fux573aux53d1ux5e03}{%

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  VERIFIED 学生创建帖子草稿。
\item
  学生选择场景并填写公共字段与场景字段。
\item
  学生提交审核，后端执行必填、长度、枚举、时间和联系方式泄露校验。
\item
  管理员审核帖子。
\item
  审核通过后帖子状态变为 PUBLISHED，并出现在广场。
\item
  审核驳回后帖子状态变为 REJECTED，学生可编辑后再次提交。
\end{enumerate}

\item

\hypertarget{ux6743ux9650ux4e0eux5b89ux5168ux8bbeux8ba1}{%
\subsection{8.

\hypertarget{ux767bux5f55ux6001ux4e0eux89d2ux8272}{%
\subsubsection{8.1
登录态与角色}\label{ux767bux5f55ux6001ux4e0eux89d2ux8272}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  用户登录后获得 JWT accessToken。
\item
  客户端在后续请求中带 Bearer Token。
\item
  后端从 token 中识别用户和 accountRole。
\item
  STUDENT 只能访问学生业务接口。
\item
  ADMIN 才能访问 \texttt{/admin/**} 接口。
\end{enumerate}

\hypertarget{ux8eabux4efdux8ba4ux8bc1ux72b6ux6001ux95e8ux7981}{%
\subsubsection{8.2
身份认证状态门禁}\label{ux8eabux4efdux8ba4ux8bc1ux72b6ux6001ux95e8ux7981}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  未登录用户只能访问健康检查、验证码、注册和登录。
\item
  已登录但未 VERIFIED 的用户不能访问广场、发帖、联系和评价等核心业务。
\item
  未认证用户访问广场列表或详情时返回
  \texttt{AUTHENTICATION\_STATUS\_REQUIRED}。
\item
  管理员审核通过身份认证后，学生才可进入主业务闭环。
\end{enumerate}

\hypertarget{ux5bc6ux94a5ux4e0eux914dux7f6eux8fb9ux754c}{%
\subsubsection{8.3
密钥与配置边界}\label{ux5bc6ux94a5ux4e0eux914dux7f6eux8fb9ux754c}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  数据库密码、JWT secret、OBS AK/SK、SMTP 授权码只存放在服务器私有
  \texttt{/etc/campus-buddy/backend.env} 或项目外私有 env。
\item
  systemd 启动脚本通过环境变量加载配置，不把 secret 放入 Java 命令行。
\item
  仓库、Markdown 文档、Qt 客户端和内测 zip 不包含真实密钥。
\item
  客户端只持有公开 API base URL，不直连数据库、不直接访问 OBS。
\end{enumerate}

\hypertarget{ux5f53ux524dux5b89ux5168ux9650ux5236}{%
\subsubsection{8.4
当前安全限制}\label{ux5f53ux524dux5b89ux5168ux9650ux5236}}

当前公网 API 仍为
HTTP，适合课程演示和内测，不适合承载真实生产数据。正式生产应接入域名、HTTPS、证书自动续期、密钥轮换、日志脱敏、备份恢复和更细权限审计。

\hypertarget{ux524dux7aefux8bbeux8ba1}{%
\subsection{9. 前端设计}\label{ux524dux7aefux8bbeux8ba1}}

\hypertarget{qt-ux684cux9762ux7aefux7ed3ux6784}{%
\subsubsection{9.1 Qt
桌面端结构}\label{qt-ux684cux9762ux7aefux7ed3ux6784}}

Qt 桌面端采用页面组件和 API service
分层。页面负责交互状态、表单校验提示和用户反馈；API service
负责构造请求、解析响应、处理业务错误；token store 负责保存当前登录态。

\hypertarget{ux4e3bux8981ux4f53ux9a8cux8bbeux8ba1}{%
\subsubsection{9.2
主要体验设计}\label{ux4e3bux8981ux4f53ux9a8cux8bbeux8ba1}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  登录注册页按``发送验证码、校验验证码、注册、登录''的自然顺序组织。
\item
  首页根据 accountRole
  动态展示学生功能或管理员功能，避免普通用户看到后台入口。
\item
  发帖页在提交审核失败时展示字段级错误，不清空用户已填内容。
\item
  广场页展示发布者认证状态和信用摘要，帮助用户判断是否联系。
\item
  会话页展示未读数、关闭状态、等待对方回复状态和联系方式解锁状态。
\item
  评价页提示 conversationId 和 revieweeId 的来源，减少人工填写错误。
\item
  关键按钮在请求中进入忙碌态，避免重复提交。
\item
  空列表、加载中、错误信息使用统一辅助样式展示。
\end{enumerate}

\hypertarget{ux5f53ux524dux9650ux5236ux4e0eux540eux7eedux6539ux8fdb}{%
\subsection{12.
当前限制与后续改进}\label{ux5f53ux524dux9650ux5236ux4e0eux540eux7eedux6539ux8fdb}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  当前公网仍为 HTTP，正式生产应接入 HTTPS、域名和证书自动续期。
\item
  投诉申诉、通知治理、用户封禁、信用人工干预等完整治理模块尚未实现。
\item
  移动端目前只有 API 支持和接入文档，正式 Android/iOS 客户端需单独开发。
\item
  登录响应中存在 refreshToken 字段，但当前后端尚未提供刷新 token 接口。
\item
  当前会话不是 WebSocket 实时聊天，主要依赖查询和刷新。
\item
  内测包适合课程演示，不等同完整商业生产系统。
\item
  服务器备份、监控告警、日志审计、密钥轮换和高并发压测仍需补齐。
\end{enumerate}

\hypertarget{ux603bux7ed3}{%
\subsection{13. 总结}\label{ux603bux7ed3}}

校园搭子平台当前实现已经形成从校园邮箱注册、身份认证、发帖、管理员审核、广场展示、低压力联系、联系方式解锁、互评信用到服务器部署和桌面端打包的最小闭环。设计重点放在校内身份可信、联系方式保护、先审后发、评价可追溯和密钥隔离上。当前版本足以支撑课程答辩和内测演示；若进入真实生产，需要继续补齐
HTTPS、治理、通知、移动端、刷新 token、监控备份和安全审计等生产化能力。
