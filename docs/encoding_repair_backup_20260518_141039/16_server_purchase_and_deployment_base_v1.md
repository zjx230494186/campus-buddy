# 16 Server Purchase And Deployment Base V1

本文记录《校园搭子平台》V1 云服务器租用与部署底座确认。当前目标不是部署完整业务系统，而是购买前确认一台后续可承载 Docker Compose、PostgreSQL、Nginx、HTTPS 验证的低成本 Linux 云服务器。

## 0. 当前状态

- 日期：2026-05-18
- 阶段：短时按需 ECS 已创建并完成 SSH 连通性确认
- 当前结论：用户选择华为云 ECS 按需计费进行短时服务器底座验证，不采用长期包年包月轻量服务器。

- 本线程不部署业务代码、不创建业务数据库表、不修改后端或 Qt 代码。

## 2.2 货比三家结论

- 纯看活动价潜力：阿里云可能最低，尤其是低至 68 元/年或 2 核 4G 9.9 元/月起的活动，但这类价格强依赖账号资格、活动库存、地域和购买页最终结算。
- 看公开价格透明度：腾讯云最好，官方价格总览直接列出规格、系统盘、带宽、流量包、月价和年付折扣，便于项目交接。
- 看同等普通入门档：腾讯云 2 核 2G 3Mbps 40 元/月与华为云 2 核 2G 2Mbps 41.62 元/月接近，但腾讯云带宽更高。
- 看部署舒适度：2 核 4G 比 2 核 2G 更适合 Java + PostgreSQL + Nginx 同机调试；若预算可接受，优先腾讯云 2 核 4G 或购买页核到更便宜的阿里云 2 核 4G。
- 当前推荐顺序：先核阿里云购买页活动价；如果阿里云 2 核 4G 年付确实低于腾讯云 2 核 2G且配置明确，则可选阿里云；否则选腾讯云 Lighthouse。

## 2.3 按需计费 ECS 判断

用户提供的华为云截图属于 ECS 自定义购买的按需计费模式，不是 Flexus L 轻量应用服务器固定套餐。

与 AutoDL 按量租用的相似点：

- 都是按使用时长计费，不需要一次性买一年。
- 适合短时间实验、临时验证、临时跑环境。
- 用完后需要主动停止或删除资源来控制成本。

与 AutoDL 无卡模式或短租算力平台的关键差异：

- 这是通用云服务器 ECS，不是 GPU 算力租赁；截图规格为通用入门型 t6.large.1，2 vCPU / 2GiB。
- 公网 IP、系统盘、带宽、流量等是独立云资源或独立计费项，不只是“机器开着才花一点钱”。
- 如果作为部署底座，需要公网 IP、系统盘、Nginx、PostgreSQL 数据目录长期存在，不能像临时训练任务那样随用随删，否则环境和数据会丢失或需要反复重建。

截图中价格估算：

```text
配置费用：0.1988 元/小时
公网出网流量：0.80 元/GB
24 小时运行：0.1988 * 24 = 4.7712 元/天
30 天运行：约 143.14 元/月，不含流量
365 天运行：约 1717.63 元/年，不含流量
```

结论：

- 如果只是 1-3 天验证 Ubuntu、Docker、Nginx、SSH、安全组，这种按需 ECS 可以用。
- 如果目标是后续持续保留部署底座、公网 IP、PostgreSQL 数据目录和 HTTPS 验证环境，不推荐按需 ECS 长期开着。
- 本项目当前更适合包年包月轻量服务器或活动价轻量服务器，因为固定成本低、交接简单、实例不会因为忘记删除而持续产生较高费用。
- 若临时使用按需 ECS，必须设置预算提醒，并在验证结束后明确删除实例、云硬盘、公网 IP 等相关资源；不得只关机后假设完全停止计费。

## 2.4 当前改用短时按需 ECS 的配置建议

用户当前计划先使用华为云 ECS 按需计费进行短时服务器底座验证。该选择可接受，但必须按“短时、可销毁、费用可控”的方式管理。

建议购买页配置：

| 配置项 | 建议值 |
|---|---|
| 计费模式 | 按需计费 |
| 区域 | 华北-北京四可接受；如后续面向南方用户或学校实际地域明确，可再调整 |
| 可用区 | 随机分配可接受 |
| 实例规格 | 2 vCPU / 2 GiB 可用于短时验证；若 Docker + PostgreSQL + 后端同时运行吃紧，再临时改 2 vCPU / 4 GiB |
| 操作系统 | Ubuntu 24.04 LTS；不可用时 Ubuntu 22.04 LTS |
| 系统盘 | 通用型 SSD 40 GiB，短时验证够用 |
| 公网 IP | 开启；按流量计费，5 Mbit/s 可接受 |
| 安全组 | 仅开放 TCP 22、80、443；PostgreSQL 5432 不对公网开放 |
| 登录方式 | SSH 密钥对，不使用长期密码登录 |
| 自动释放/费用提醒 | 能配置则必须配置；验证结束后删除实例、云硬盘和公网 IP 等关联资源 |

密钥配置原则：

- 不在聊天或项目文档中粘贴私钥内容。
- 优先使用本地生成密钥对并在华为云导入公钥；私钥只保存在用户本机。
- 如果使用华为云控制台创建密钥对，下载得到的私钥也只保存在用户本机，不上传到仓库，不写入文档。
- 后续需要由 Codex 登录服务器时，只提供本机私钥文件路径和公网 IP，不提供私钥明文。
- 服务器验证完成后，可在华为云删除该临时密钥对，或在服务器上移除对应公钥。

推荐本地密钥路径示例：

```text
C:\Users\zjx230494186\.ssh\campus_huawei_ecs_20260518
C:\Users\zjx230494186\.ssh\campus_huawei_ecs_20260518.pub
```

后续登录命令形态：

```powershell
ssh -i "$env:USERPROFILE\.ssh\campus_huawei_ecs_20260518" root@<公网IP>
```

为什么适合当前阶段：

- 2 核 2 GB 是 Docker Compose + Nginx + Spring Boot 探路 + PostgreSQL 自建的最低可用档；能跑最小闭环，但不适合高并发或重负载。
- 40 GB 系统盘足够当前部署骨架、容器镜像、少量日志和测试数据库；后续需要明确日志轮转与数据库备份。
- 3 Mbps / 200 GB 月流量足够课程项目、低访问量联调和 HTTPS/Nginx 验证。
- 腾讯云官方价格表清晰，便于交接和预算复核。
- 后续如 2 GB 内存不足，可保留迁移路径：升级到 2 核 4 GB，或将 PostgreSQL 迁移到独立数据库/RDS。

## 5. 备案、实名、优惠判断

实名：

- 中国大陆云厂商购买云服务器通常需要账号实名认证。
- 腾讯云官方文档明确：购买云服务器实例之前需要实名认证。

备案：

- 如果使用中国内地服务器，并用域名对外提供网站或 App 服务，需要 ICP 备案。
- 如果当前仅通过公网 IP 做 SSH、Docker、Nginx 健康检查或内部联调，不绑定域名对外提供正式网站，仍可先完成服务器底座验证；后续绑定域名和开放正式访问前必须处理备案。
- 若要走腾讯云备案，轻量应用服务器需为中国境内、包年包月、购买 3 个月及以上，且备案期间剩余有效期需大于等于 1 个月。

学生优惠/新人优惠：

- 可以在购买页核对新人、学生、活动价，但不把活动价写成长期稳定成本。
- 若腾讯云/阿里云购买页出现低于官方常规价的活动价，可优先使用，但必须确认续费价、活动限制和是否影响升级。
- 本文预算采用腾讯云官方常规价格表估算，避免把抢购价作为交付基线。

## 6. 购买前确认清单

付款或开通前逐项确认：

- 已登录用户自己的云账号，且账号已完成实名认证。

- 确认产品：轻量应用服务器 Lighthouse，不选择轻量数据库、云数据库、对象存储或额外安全增值包。
- 确认地域：中国内地，优先上海。

### 7.2.3 Windows OpenSSH 私钥权限

Windows OpenSSH 会拒绝使用权限过宽的私钥文件，常见错误如下：

```text
WARNING: UNPROTECTED PRIVATE KEY FILE!
Permissions for '<key>.pem' are too open.
This private key will be ignored.
```

本次处理方式：

```powershell
icacls "C:\Users\zjx230494186\.ssh\campus-huawei-ecs-20260518-BigHomework.pem" /inheritance:r
icacls "C:\Users\zjx230494186\.ssh\campus-huawei-ecs-20260518-BigHomework.pem" /grant "DESKTOP-D5NFUN8\zjx230494186:R"
```

处理后检查：

```powershell
icacls "C:\Users\zjx230494186\.ssh\campus-huawei-ecs-20260518-BigHomework.pem"
```

预期只保留当前用户读取权限。若后续用户名或主机名变化，需要用 `whoami` 获取当前账户名后重新授权。

### 7.2.5 常见问题与排查

| 现象 | 常见原因 | 处理 |
|---|---|---|
| `Connection timed out` | 安全组未开放 22、实例未开机、公网 IP 错误、本地网络阻断 | 先在控制台确认实例运行中和公网 IP，再用 `Test-NetConnection -ComputerName 114.116.203.78 -Port 22` 检查端口 |
| `TcpTestSucceeded: False` | 22 端口未通 | 检查安全组入方向、实例防火墙、EIP 绑定状态 |
| `UNPROTECTED PRIVATE KEY FILE` | PEM 文件权限过宽 | 按 7.2.3 收紧 Windows ACL |
| `Permission denied (publickey,password)` | 私钥不匹配、用户名错误、实例未绑定该密钥 | 确认实例绑定的密钥对、私钥路径、用户名 `root`，必要时通过云控制台重置密钥 |
| 首次连接提示确认指纹 | 本机尚未信任该主机 | 确认 IP 是当前实例公网 IP 后接受；命令中可用 `StrictHostKeyChecking=accept-new` |
| Linux 命令里误用 `Select-String` | `Select-String` 是 PowerShell 命令，不是 Ubuntu 命令 | 远端命令使用 `grep`、`awk`、`sed` 等 Linux 工具 |
| 后续 HTTP/HTTPS 无法访问 | 80/443 未开放、Nginx 未安装或未监听、云防火墙阻断 | 先确认安全组，再在服务器上检查 `ss -tulpn` 和 Nginx 状态 |
