# 19 公网 IP 开发演示访问决策 V1

本文记录《校园搭子平台》V1 在当前开发与课程演示阶段的后端访问入口决策，用于修正早期沟通中可能让人误以为“必须立即申请域名与 HTTPS 证书”的表述。

## 1. 当前决策

- 当前开发与软件演示阶段：直接使用 ECS 公网 IP 作为后端访问入口。
- 当前不申请域名。
- 当前不申请 HTTPS 证书。
- Qt PC 客户端通过可配置的 API base URL 访问后端；开发/演示阶段可配置为公网 IP。
- 域名、备案、HTTPS 证书不作为当前开发、部署骨架验证或课程演示的阻塞项。

## 2. 决策理由

- 本项目首版交付形态是 Win11 PC 端 Qt 软件，演示观众主要看到客户端界面，不会像浏览器一样直接看到地址栏、锁标识或证书详情。
- 当前目标是完成最小可执行、可验证、可交接的工程闭环；公网 IP 已满足后端 API 连通、Qt 客户端联调、Nginx 反向代理和对象存储服务端访问验证。
- 域名和 HTTPS 主要提升生产级安全性、稳定性、迁移便利性和正式对外交付规范性，但不是当前开发演示的硬件前置要求。

## 3. 对旧结论的修正

以下旧表述需要按本决策理解：

- “Nginx + HTTPS 方案”“Certbot / 云厂商免费证书”“申请证书前确认域名、备案和 DNS”仍可作为未来生产化或正式上线前的备选工作，但不再代表当前阶段必须执行。
- “部署骨架验证：Compose Nginx HTTPS”调整为“部署骨架验证：Compose Nginx 公网 IP HTTP；HTTPS 仅保留为上线前增强项”。
- 当前可以先验证 HTTP 健康检查、后端 API 反向代理、Qt 客户端到后端 API 的 HTTP/JSON 连通性。
- 若未来处理真实用户密码、token、认证材料或长期公网服务，仍建议切换到域名 + HTTPS；该建议不影响当前课程演示阶段使用公网 IP。

## 4. 当前配置边界

| 项目 | 当前决策 |
|---|---|
| 后端访问入口 | ECS 公网 IP |
| Qt 客户端 API base URL | 使用可配置项，当前可填公网 IP |
| 域名 | 当前不申请 |
| HTTPS 证书 | 当前不申请 |
| Nginx | 可继续用于公网 IP HTTP 反向代理与健康检查 |
| 备案 | 当前不处理；未来绑定中国内地域名并对外正式提供服务前再评估 |
| 生产安全建议 | 保留域名 + HTTPS 作为上线前检查项 |

## 5. 后续实现注意

- Qt 客户端不要把公网 IP 写死在深层业务代码中，应保留 API base URL 配置入口。
- 当前演示环境可以使用 HTTP；文档和答辩说明中应标注为开发/演示环境。
- 若后续进入真实用户试用或公网长期运行，需重新评估 HTTPS、域名、备案、证书续期和安全加固。
- 本决策不改变对象存储边界：Qt 客户端仍不得持有 OBS 凭证，也不得直连 OBS。
