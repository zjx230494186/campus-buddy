## 1. 上游约束

来自 `docs/13_detailed_design_v1.md` 的对象存储技术约束：

- 文件/附件存储架构采用 S3 兼容对象存储接口抽象。
- 生产候选优先评估阿里云 OSS、腾讯云 COS、华为云 OBS。
- MinIO 作为本地开发、测试或私有化部署备选。
- 附件访问控制由后端业务权限校验完成。
- V1 不让客户端直传对象存储。
- V1 采用后端中转上传对象存储。
- PostgreSQL 只保存附件元数据、业务归属、权限、审核状态、对象 key、哈希值、大小和 MIME 类型。

本线程额外限制：

- 不实现业务附件上传接口。
- 不创建业务数据库表。
- 不修改后端或 Qt 代码。
- 不让 Qt 客户端或用户侧直接持有对象存储访问凭证。
- 不创建超出对象存储确认所需的长期付费附加云产品。

## 4. 推荐决策包

推荐方案：华为云 OBS，标准存储，私有桶，后端中转访问。

| 配置项 | 推荐值 |
|---|---|
| 云厂商 | 华为云 |
| 产品 | 对象存储服务 OBS |
| 地域 | 优先 `华北-北京四`，与当前 ECS 保持一致 |
| 存储类型 | 标准存储，单 AZ 优先 |
| 计费方式 | 初期按需计费，不先购买资源包；若后续附件量稳定再评估标准存储包或流量包 |
| 桶权限 | 私有；不开启公共读、不开启公共读写 |
| 公网读 | 默认不允许；不暴露 OBS 裸 URL 给 Qt 客户端或未授权用户 |
| 服务端访问 | 后端服务通过服务端凭证访问 OBS；优先从同区域 ECS 使用内网 Endpoint 或内网访问方式 |
| 客户端访问 | Qt 客户端只访问后端 API，不持有 OBS AK/SK、临时凭证或桶直连地址 |
| CORS | 默认不配置；只有后续 Web 管理端或浏览器直连对象存储场景明确成立时再单独评估 |
| AccessKey 管理 | 不使用主账号长期密钥；后续仅为后端服务创建最小权限 IAM 用户或等价授权；密钥只进入服务器本地环境变量、密钥管理服务或部署私有配置，不入库、不入文档 |
| 对象 key 命名 | 由后端生成，不信任客户端文件名；建议前缀按业务域划分，如 `auth-materials/`、`post-attachments/`、`case-evidence/`，再按日期和随机 ID 组织 |
| 删除策略 | 认证材料审核完成或驳回后按详细设计删除原件或解除业务引用；测试桶不用后删除对象和桶 |

## 5. 桶命名规则建议

OBS 桶名需全局唯一，实际创建时可以在以下规则基础上追加短随机后缀：

```text
campus-partner-v1-dev-cn-north-4-<short-suffix>
campus-partner-v1-test-cn-north-4-<short-suffix>
campus-partner-v1-prod-cn-north-4-<short-suffix>
```

当前阶段建议只创建开发/验证桶：

```text
campus-partner-v1-dev-cn-north-4-<short-suffix>
```

注意：

- `<short-suffix>` 不记录账号敏感信息。
- 不把学号、手机号、邮箱、真实姓名等个人信息写入桶名或对象 key。
- 生产桶、测试桶、开发桶应分离，避免测试清理误删生产对象。

## 6. 权限策略结论

当前推荐权限边界：

- 桶级别保持私有。
- 禁止公共读写。
- 默认不开放公共读。
- 后端服务账号只授予当前桶必要的对象读、写、删除和有限列表权限。
- 不给 Qt 客户端发放 AK/SK 或临时凭证。
- 不在数据库、Markdown、聊天、代码注释、测试记录中写入 AK/SK 明文。
- 后续正式实现前，应通过配置体系明确以下敏感项只存在于部署私有配置中：
  - `OBJECT_STORAGE_ENDPOINT`
  - `OBJECT_STORAGE_REGION`
  - `OBJECT_STORAGE_BUCKET`
  - `OBJECT_STORAGE_ACCESS_KEY_ID`
  - `OBJECT_STORAGE_SECRET_ACCESS_KEY`

可选增强：

- 如果后端部署在华为云 ECS，优先探索 IAM 委托或等价方式，减少长期 AK/SK 暴露面。
- 如果必须使用 AK/SK，使用专用 IAM 用户、最小权限、定期轮换，并保留密钥撤销流程。

## 7. CORS 结论

当前 V1 Qt 桌面端和后端中转模式下，浏览器跨域不是必要条件，因此默认不配置 OBS CORS。

后续只有在以下情况出现时，才重新开启 CORS 设计：

- 新增 Web 管理端直接从浏览器访问对象存储。
- 改为浏览器直传对象存储。
- 使用对象存储静态网站或自定义域名承载公开资源。

即使未来配置 CORS，也不得绕过后端业务权限校验；CORS 只解决浏览器同源策略，不等同于授权。

## 8. 服务端中转访问方式

推荐访问链路：

```text
Qt 客户端 -> 后端 REST API -> ObjectStorageService 抽象 -> 华为云 OBS
```

上传链路：

1. Qt 客户端把文件提交到后端业务接口。
2. 后端校验登录态、业务权限、文件类型、大小、MIME、哈希等。
3. 后端生成对象 key。
4. 后端写入 OBS 私有桶。
5. 后端保存附件元数据到 PostgreSQL。
6. 后端只返回业务附件 ID 或受控访问入口，不返回 OBS 裸 URL。

下载/预览链路：

1. Qt 客户端请求后端附件访问接口。
2. 后端校验登录态、业务归属和附件状态。
3. 后端从 OBS 读取对象流。
4. 后端把内容以受控响应返回客户端。

当前线程不实现以上接口，只确认底座口径。

## 9. 后续迁移路径

迁移设计原则：

- 数据库只保存对象 key、桶逻辑标识、大小、哈希、MIME、状态和业务归属，不保存厂商裸 URL 作为业务主键。
- 后端通过 `ObjectStorageService` 或等价接口隔离厂商 SDK。
- 配置层区分 `provider`、`endpoint`、`region`、`bucket`、`pathStyle` 或等价兼容参数。
- 对象 key 规则与厂商无关。

从 OBS 迁移到 OSS/COS/MinIO 时，理论步骤：

1. 在目标对象存储创建私有桶。
2. 用迁移工具或脚本复制对象。
3. 保持对象 key 不变。
4. 切换后端对象存储 provider 和 endpoint 配置。
5. 做上传、读取校验和删除校验。
6. 确认无误后清理旧桶对象。

## 10. 购买 / 开通前确认清单

用户在华为云控制台真实开通前，应逐项确认：

- 只开通对象存储 OBS，不额外开通 CDN、图片处理、数据处理、日志分析、备份、安全增值包等长期付费附加项。
- 地域选择 `华北-北京四`，除非控制台价格或账号限制要求调整。
- 桶为标准存储、私有权限。
- 不开启公共读、公共读写。
- 不把 AccessKey、SecretKey、临时凭证粘贴给 Codex 或写入项目文档。
- 如果创建 IAM 用户或访问密钥，只授权当前桶所需最小权限。
- 开通后记录：云厂商、产品、地域、桶名、权限策略结论、计费方式、清理提醒。
- 若只是验证，验证结束后删除桶内对象并删除桶，避免继续产生存储和请求等费用。
